Порівняння ефективності SQL та ORM-запитів (trips_db)
Даний проєкт присвячений дослідженню продуктивності взаємодії з реляційною базою даних MySQL за допомогою чистого SQL та ORM SQLAlchemy. Проєкт розроблено в межах курсової роботи студента групи ФЕП-11 Рибака О.В.

1. Опис проєкту
Система trips_db моделює роботу туристичної компанії (бронювання поїздок, логістика, оплати). Мета дослідження: Експериментально порівняти швидкість виконання запитів (JOIN, агрегація) при використанні різних підходів та оцінити вплив механізму FOREIGN KEY на продуктивність системи.

2. Вимоги до середовища
Python 3.10+

MySQL Server 8.0+

Бібліотеки: sqlalchemy, pymysql, matplotlib, pandas.

3. Налаштування бази даних
Інструкція, як імпортувати структуру та дані:

Bash

mysql -u root -p < database/trips_db.sql
4. Встановлення залежностей
Bash

pip install -r requirements.txt
5. Порядок проведення експерименту
Для відтворення результатів дослідження та замірів часу виконуйте скрипти у наступному порядку:

Перевірити підключення до бази:

Bash

python test_connection.py
Створити таблиці:

Bash

python create_tables.py
Додати тестові дані:

Bash

python insert_data.py
Перевірити ORM-запити (зі зв’язками):

Bash

python queries.py
Перевірити SQL-запити (зі зв’язками):

Bash

python queries_sql.py
Перевірити запити без зв’язків:

Bash

python queries_no_fk_orm.py
python queries_no_fk_sql.py
6. Графічний клієнт
Для візуальної взаємодії з базою даних запустіть:

Bash

python client_app.py
7. Особливості реалізації
Проблема N+1: У коді продемонстровано, як автоматична навігація ORM по зв'язаних об'єктах призводить до значних затримок у порівнянні з прямими SQL-запитами.

Гнучкість архітектури: Реалізовано можливість тестування як на базі з активованими FOREIGN KEY, так і на базі з ручним приєднанням таблиць (без зв'язків), що дозволяє оцінити накладні витрати на підтримку цілісності даних.
